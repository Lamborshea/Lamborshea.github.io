修改Idea终端Terminal为GitBash或Cmder

2018-11-11

文章目录

  1. 修改 Idea 终端 Terminal 为 GitBash
  2. 修改 Idea 终端 Terminal 为 Cmder

修改 Idea 终端 Terminal 为 GitBash

  1. 打开设置（快捷键：Ctrl + Alt + S），进入 Plugins,搜索栏搜索 Terminal，查看 Terminal插件是否打勾选中，如果没有，请打勾。
  2. 进入设置（快捷键：Ctrl + Alt + S），进入 Tools字段，再进入 Terminal字段，在 Shell path那一栏中，输入你主机 Git Bash的安装位置
  1. 重新启动 Idea IDE，然后打开 Terminal 查看是否配置正确。


如果没有，请检查路径中，是否正确加了英文双引号，路径是否正确。

修改 Idea 终端 Terminal 为 Cmder

  1. 增加系统变量
    变量名为：CMDER_ROOT
    变量值为：C:\Application\cmder (主机上 Cmder 安装主目录)
  2. 打开设置（快捷键：Ctrl + Alt + S），进入 Plugins,搜索栏搜索 Terminal，查看 Terminal插件是否打勾选中，如果没有，请打勾。
  3. 进入设置（快捷键：Ctrl + Alt + S），进入 Tools字段，再进入 Terminal字段，在 Shell path那一栏中，输入你主机 Git Bash的安装位置
  1. 重新启动 Idea IDE，然后打开 Terminal 查看是否配置正确。

    如果没有，请检查路径中，是否正确加了英文双引号，路径是否正确。有的老版本，cmd.exe两边是不需要加双引号的，但是新版本的策略中，都要求加双引号了。
